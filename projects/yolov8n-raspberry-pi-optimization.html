Project Evolution Journey

This project represents a comprehensive journey in model optimization and edge deployment, guided by Prof. Narayan Sir (HOD, ECE Department). Starting with RT-DETR achieving excellent accuracy on laptops, we encountered significant performance challenges when deploying to Raspberry Pi 4, leading to an extensive research and optimization process.

Research Problem Statement

Challenge: RT-DETR model worked perfectly on laptops but suffered severe performance degradation on Raspberry Pi 4 (4-core CPU limitation). This led us to explore YOLO architectures and advanced edge optimization techniques to achieve real-time performance while maintaining detection accuracy.

Model Evolution Timeline

1

RT-DETR Initial Implementation

Successfully trained RT-DETR on 25,000 head images achieving 98% accuracy on laptops. However, deployment on Raspberry Pi 4 resulted in severe FPS drops due to GPU-optimized architecture incompatibility with 4-core CPU.

Issue: RT-DETR requires GPU cores for optimal performance, causing lag on Raspberry Pi's limited CPU architecture.

2

YOLO Architecture Exploration

Under Prof. Narayan Sir's guidance, we explored YOLO models known for Raspberry Pi compatibility. Trained both YOLOv8n and YOLOv11n on Kaggle (6 hours each) with 30,000 images dataset.

Edge Optimization Research

Extensive research into edge computing techniques for model compression and optimization. Tested multiple conversion formats to find the optimal balance between speed and accuracy.

Techniques Tested

Counting Accuracy Pipeline Enhancement

False Count Detection & Solution

Advanced Pipeline Logic for Accurate Counting

During Raspberry Pi testing at 12 FPS, we identified a critical issue where the system was generating unnecessary fake counts. The problem occurred when heads were detected but the person wasn't actually crossing the counting line. We developed an enhanced pipeline that implements a "suspense" mechanism to ensure accurate counting.

Before & After: Counting Accuracy Improvement

✅ Solution: Real Counts

Enhanced pipeline with accurate counting logic

Improvements Made:

  • • Count only when person completely exits frame

Counting Logic & Real-world Challenges

Smart Counting Algorithm

Net Passenger Counting Logic

Handling Peeping Head & Duplicate Count Issues

Observed Challenge
  • • Peeping Head Issue: Person peeps head out and pulls back
  • • Result: IN and OUT count increases repeatedly
  • • Track ID: Remains same (Track = 1)
  • • Problem: Entry/exit line crossing triggers multiple counts
Smart Solution
  • • Net Count Logic: IN_count - OUT_count
  • • Benefit: Even with peeping, net count remains correct
  • • Use Case: Perfect for bus passenger occupancy
  • • Result: Accurate passenger estimation
Counting Algorithm
IN_count
People entering bus
OUT_count
People exiting bus
Net = IN - OUT
Current occupancy

✓ Solution Effectiveness: This net counting logic is sufficient for bus passenger occupancy estimation, which is the primary use case. Even if a person peeps and causes simultaneous IN and OUT counts, the net passenger count remains accurate for fleet management purposes.
